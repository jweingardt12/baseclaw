How should we organize the expanded dashboard?

You currently have 6 pages: Today, Roster, Matchup, Players, League, Settings. We need to fit ~15 new feature areas.

A

Enrich Existing Pages

Keep the same 6 pages but add tabs/sections within them. Today gets games strip + news + streaming recs. Matchup gets category sim + trade tools. Players gets waiver analysis tab. League gets power rankings.

Pros

  • Simple navigation
  • Related data stays together
  • Less context switching

Cons

  • Pages get heavy/long
  • Some features feel buried
B

Add 2-3 New Pages

Keep existing pages + add a "Strategy" page (trade finder, category sim, punt advisor, playoff planner) and a "Intel" page (news, breakouts, probable pitchers, closer monitor, streaming). Enrich existing pages with the obvious fits.

Navigation
Today Roster Matchup Players League Strategy ✨ Intel ✨ Settings

Pros

  • Clear feature grouping
  • Dedicated space for advanced tools
  • Mobile-friendly (5 bottom tabs + overflow)

Cons

  • More nav items
  • Need to decide on grouping
C

Hub + Spoke (Smart Today Page)

Make "Today" a rich command center that surfaces the most important data from everywhere. Other pages become detail views you drill into. Today shows: games strip, matchup score, action items, waiver recs, streaming picks, news headlines, injury alerts — all in a dense card grid.

Pros

  • One-stop overview
  • Most actionable layout for daily use
  • Matches how fantasy managers actually check their team

Cons

  • Today page becomes complex
  • Still need detail pages
